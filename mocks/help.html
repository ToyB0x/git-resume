ヘルプ & サポート

Deep Researchの基本的な使い方

1

GitHub User名を入力

ホーム画面でGitHub User名を入力し、「調査を開始」ボタンをクリックします。

ヒント: 公開プロフィールを持つGitHubユーザー名であれば誰でも分析できます。

2

調査計画を確認

入力したGitHub User名の基本情報と調査計画が表示されます。内容を確認し、「調査を実行」ボタンをクリックして詳細分析を開始します。

ヒント: この段階では軽量な情報のみを取得しているため、10秒以内に結果が表示されます。

3

調査の進捗を確認

詳細分析の進捗状況がリアルタイムで表示されます。この処理はバックグラウンドで実行されるため、いつでも安全に画面を離れることができます。

ヒント: 分析にはリポジトリ数に応じて数分から数十分かかる場合があります。

4

結果を確認

分析が完了すると、マークダウン形式のレジュメが表示されます。「エクスポート」ボタンでPDFやテキスト形式で保存したり、「共有」ボタンで結果を共有することができます。

ヒント: 分析結果は30日間保存され、同じユーザー名で再検索すると即座に表示されます。

2段階調査プロセスについて

1次調査（調査計画段階）

GitHub User名を入力してから10秒以内に基本的なプロフィール情報を取得し、調査計画を生成します。この段階では軽量な処理のみを行い、キャッシュは使用しません。

2次調査（詳細分析段階）

「調査を実行」ボタンをクリックすると、詳細なGitHub活動の分析とレジュメ生成を行います。この処理は重いため、バックグラウンドで実行され、結果は30日間キャッシュされます。

  • リポジトリの検索: 過去1年間にコミットしたリポジトリを検索
  • リポジトリのClone: 検索結果に基づきリポジトリをClone
  • リポジトリの活動分析: ユーザーのコミットのみを抽出し分析
  • Resumeの作成: 分析結果からマークダウン形式のResumeを生成
